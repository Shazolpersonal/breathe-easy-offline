## 2023-10-27 - Map/Set Lookups in useMemo
**Learning:** Using `Array.prototype.find()` or `Array.prototype.includes()` inside `useMemo` hooks that iterate over large arrays (like technique lists) creates an $O(N^2)$ or $O(N \times M)$ bottleneck, causing performance degradation as datasets grow. Also, dependency array optimization should not break data fetching logic; sometimes dependencies correctly trigger updates even if not explicitly used in calculations (like fetching fresh `progressions` from local storage when custom techniques are added).
**Action:** When working with large datasets in `useMemo`, map secondary data (like favorites or progressions) into Sets or Maps *before* iterating to convert inner searches into $O(1)$ lookups.
